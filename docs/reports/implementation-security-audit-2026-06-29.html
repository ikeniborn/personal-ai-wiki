Выявленные проблемы

IDSeverityПроблемаДоказательствоРекомендация
SEC-01HIGH HTML GET-маршруты авторизуют по наличию session id, не проверяя существование пользователя. src/paw/api/web/routes.py:173 возвращает uid из Redis; dashboard, domain_page, article_page, query_page и др. проверяют только uid. Если пользователь удален, его Redis-сессия до TTL продолжает читать web-страницы. API dependency current_user такой проблемы не имеет. Заменить проверки _current_uid на обязательный helper, который загружает User, удаляет/игнорирует stale session при отсутствии пользователя и применяет role check для read pages.
SEC-02HIGH Upload endpoints читают весь request body в память до проверки лимитов. src/paw/api/routers/sources.py:39 и :63 вызывают await file.read(); validate_source_upload проверяет размер уже после чтения. max_request_bytes задан в src/paw/config.py:34, но не используется. Добавить middleware/ASGI guard по Content-Length и streaming read с hard cap. Для multipart ограничить размер на уровне reverse proxy и приложения, не полагаясь только на post-read validation.
SEC-03MED Docker build не воспроизводит lockfile, несмотря на наличие uv.lock. Dockerfile:5-9 копирует только pyproject.toml и запускает uv pip install --system .. В pyproject.toml зависимости заданы lower-bound диапазонами, значит production image может получить версии, отличающиеся от CI/local lock. Копировать uv.lock и ставить зависимости через locked/frozen workflow, например uv sync --frozen --no-dev или эквивалентный locked install для image.
SEC-04MED SSRF-защита валидирует DNS отдельно от фактического подключения. src/paw/security/ssrf.py:52 проверяет socket.getaddrinfo, затем httpx на :70 заново резолвит адрес при запросе. При DNS rebinding между check/use возможно подключение к внутреннему адресу. Пиннить проверенный IP в transport/connection layer или использовать resolver/transport, который подключается к уже проверенному адресу с сохранением Host/SNI. Повторять проверку для каждого redirect уже делается, но TOCTOU остается.
SEC-05MED /metrics открыт без app-level auth. src/paw/main.py:72 публикует метрики всем, кто достиг API. Traefik rule в docker-compose.yml:88-91 маршрутизирует весь host на API без path deny/basic auth для /metrics. Ограничить /metrics отдельной internal-only сетью, auth middleware, Traefik middleware по IP allowlist или убрать endpoint из публичного router.
SEC-06MED Нет rate limiting для login/setup и нет минимальной password policy. src/paw/api/routers/auth.py:21 и src/paw/api/routers/setup.py:86 принимают пароли без throttling/lockout; src/paw/security/passwords.py надежно хеширует Argon2, но не валидирует силу пароля. Добавить Redis-backed throttling по IP/email, задержку на неверный login, минимальную длину и проверку слабых паролей для setup/users.
SEC-07LOW Audit log объявлен, но не подключен к чувствительным операциям. Есть модель AuditLog и helper src/paw/audit/log.py, но grep не нашел вызовов record(...) в сервисах/API. Логировать login/logout, setup, user/role changes, API key issue/revoke, provider settings changes, ingest/rollback/delete operations.
OPS-01MED Compose defaults допускают слабый Postgres password. docker-compose.yml:34, :157, :189 используют fallback POSTGRES_PASSWORD:-paw; .env.example:19 также содержит POSTGRES_PASSWORD=paw. Убрать production fallback, требовать explicit secret, добавить fail-fast проверку в compose/init docs.
FUNC-01MED Bulk upload UI ожидает HTML drawer, но вызывает JSON API. src/paw/api/web/templates/domain.html:12-14 отправляет HTMX в /api/v1/.../sources/bulk с hx-target="#job-drawer". Endpoint src/paw/api/routers/sources.py:52 возвращает JSON BulkOut, не _job_drawer.html. Сделать web endpoint для bulk upload, который возвращает drawer/список jobs, либо изменить UI на обработку JSON и отдельный запуск drawer.
QUAL-01LOW В репозитории есть generated bytecode вне .venv. find обнаружил __pycache__/*.pyc в src/, tests/, alembic/. Они игнорируются git, но засоряют рабочее дерево и могут попадать в сторонние архивы, если не использовать .dockerignore. Очистить рабочее дерево командой удаления pycache перед упаковкой/релизом; оставить ignore rules.

Зависимости

  • pip-audit: известных уязвимостей в установленной среде не найдено.
  • Основной supply-chain риск не CVE, а разрыв между uv.lock и Docker build.
  • В CI используется uv sync --dev, а Dockerfile делает отдельный resolve/install из pyproject.toml.
  • Dev tooling временно запускался через uv run --with ..., проектные зависимости не менялись.
