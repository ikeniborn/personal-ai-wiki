Критерии успеха (Acceptance, проверяемые)

#Критерий
1/metrics отдаёт api RED + worker arq + domain/token/cost счётчики; /health отдельный, рапортует readiness.
2docker compose --profile observability up поднимает Prometheus + Grafana + экспортёры с рабочими дашбордами; без профиля приложение работает без изменений.
3Langfuse off by default; при включении op даёт trace с per-LLM generation-спанами; падение Langfuse не валит op.
4pg_dump бэкап идёт по расписанию; восстановление из дампа воспроизводит корпус.
5URL-loader отклоняет private/link-local/non-https цели + oversize-тела; zip-bomb отклонён; не-allowlisted типы загрузок отказаны.
6epub, url, image (OCR) источники ингестятся в статьи; bulk zip регистрирует + ингестит несколько источников.
7Переключатель языка UI меняет RU/EN независимо; api-keys и users управляемы в admin UI.

Тесты

  • Unit: SSRF-решения, anti-zip-bomb, magic-byte, кардинальность меток метрик, Langfuse no-op при выключении.
  • Integration (testcontainers): контент /metrics, roundtrip backup→restore, epub/url/image loaders со стабами, bulk-zip ingest.
  • API: переключатель i18n, управление api-keys/users, /health vs /metrics.
  • E2E: профиль observability up + scrape, ingest url/epub/image, backup/restore.
