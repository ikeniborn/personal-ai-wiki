IDSeverityСекцияОписаниеVerdict
F-001WARNINGT4 / ChunkRepo.set_embedding Литерал вектора строится через repr(float(x)); repr может дать научную нотацию (1e-05) или inf/nan, которые текстовый парсер pgvector способен отвергнуть/исказить. Нужна защита формата или отбраковка не-конечных значений. open
F-002WARNINGT5 / append_log В блоке кода показан inline-шим __import__("json"), а затем прозой — инструкция заменить его на верхнеуровневый import json. Дословное копирование блока worker'ом протащит шим; каноничным должен быть только финальный вид кода. open
F-003WARNINGT5 / reconcile_stuck reconcile_stuck(older_than_seconds=0) ловит свежезапущенный job через heartbeat_at IS NULL. Но при now() - 0 == now() совпадёт и job, успевший сделать heartbeat; граничная семантика (≥ vs >) не покрыта тестом и может зареконсайлить живой job. open
F-004WARNINGGlobal Constraints / именование индексов Сказано, что явные имена CREATE INDEX «должны соответствовать стилю ix_%(column_0_label)s где практично», но ручные имена (ix_chunks_tsv и т.п.) — простой ix_<table>_<col>, не label-форма. Безвредно (raw SQL обходит convention), но заявленное «must-match» фактически не проверяется. open
