1. Резюме спецификации

Каждое требование — отдельной строкой; секции спеки соответствуют покрываемым задачам.

СекцияТребование
§1. System summarySelf-hosted RAG-вики командного масштаба: загрузка источников (md/pdf/docx/html/epub/url/images) → LLM-харнесс извлекает темы и генерирует статьи с сущностями, ссылками и цитатами.
§1. System summaryГибридный поиск (вектор + FTS + graph BFS); режимы query (одноразовый, кэшируемый) и chat (тредовый) по домену с цитируемыми ответами; редактирование статей с историей ревизий; граф-вью; lint/fix/format/reindex; read-only MCP-эндпоинт.
§2. StackPython 3.12 · uv · FastAPI (async) · Jinja2 + HTMX + Cytoscape.js (vendored) · PostgreSQL 16 + pgvector · Redis + arq · OpenAI-совместимый LLM · MCP Python SDK (Streamable HTTP) · Docker Compose + Traefik.
§3. Module layoutПакет paw, src-layout, один образ / два процесса (api uvicorn, worker arq), MCP смонтирован в приложение. Полное целевое дерево — в LLD §1.
§3. Dependency rulesПравила зависимостей без циклов: api/web/mcp → services + retrieval; services → db.repos, storage, vector, graph, jobs, harness; harness → providers (storage/vector/graph через tools); ingest/vector/graph/storage/providers → db; db, config — листья.
§4. Phasing strategyWalking skeleton + вертикальный рост: Фаза 1 — тонкий end-to-end срез; каждая следующая фаза добавляет одну реальную возможность end-to-end. Фазы ≠ секции LLD (LLD §1–§13 — кросс-срезы дизайна, не порядок сборки).
§5. Phase mapТаблица из 9 фаз: «значимость · зависимости · имя spec-файла». Рекомендуемый порядок 1→2→3→(4,5,6 параллельно)→7→8→9; Фаза 9 — последняя.
§6. Global UI decisionsРешения, обязательные для каждой фазы с UI: двойной сайдбар (wiki-style frame), domain landing, страница статьи (Edit/Preview, 409-баннер), Query (SSE + stale badge), Chat (история сессий), Graph (full-canvas Cytoscape + drawer), Settings (single page), Jobs drawer (SSE), setup wizard, UI i18n.
§7. Cross-cutting conventionsСквозные конвенции: config layering (LLD §10); secrets (argon2, hashed API-keys, Fernet-encryption); API-конвенции (LLD §8: сессии в Redis, RBAC, CSRF, RFC 9457, keyset-пагинация, SSE); harness safety (LLD §4: tool-allowlist, write-scope, schema-validation, лимиты, audit_log); фиксированный embedding dim; тестирование (LLD §11); observability.
§8. DeferredBacklog (LLD §13): rate limiting, idempotency keys, soft-delete, reranking, resumable ingest, graph pagination, scheduled jobs. Out of v1: per-domain ACL, config hot-reload, webhooks, eval-харнесс.
§9. How to consumeКаждый phase-spec самодостаточен и готов к writing-plans: Фаза 1 → writing-plans → реализация → verify acceptance → следующая фаза. Спеки ссылаются на LLD, не дублируя DDL/контракты/пайплайны.
