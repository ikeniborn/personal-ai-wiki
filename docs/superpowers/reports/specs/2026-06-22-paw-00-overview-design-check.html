Каждое требование — отдельной строкой; источник — секция спеки. Это индекс-спека: она не дублирует LLD, а ссылается на §N и делит сборку на фазы.

#СекцияТребование
R1§1 System summarySelf-hosted team-scale RAG-вики: загрузка источников (md/pdf/docx/html/epub/url/images) → LLM-харнес генерирует статьи с сущностями/ссылками/цитатами; гибридный retrieval (vector + FTS + graph BFS); query (кэш) / chat (треды); редактирование с историей; граф-вид; lint/fix/format/reindex; read-only MCP.
R2§2 StackPython 3.12 · uv · FastAPI(async) · Jinja2+HTMX+Cytoscape.js(vendored) · PostgreSQL 16+pgvector · Redis+arq · OpenAI-совместимый LLM · MCP Python SDK (Streamable HTTP) · Docker Compose + Traefik.
R3§3 Module layout (LLD §1)Пакет paw, src-layout, один образ / два процесса (api uvicorn, worker arq), MCP вмонтирован. Правила зависимостей без циклов: api/web/mcp→services+retrieval; services→repos/storage/vector/graph/jobs/harness; harness→providers и storage/vector/graph через tools; ingest/vector/graph/storage/providers→db; db,config — листья.
R4§4 Phasing strategyWalking-skeleton + vertical growth: фаза 1 — тонкий end-to-end срез; каждая следующая добавляет одну реальную возможность end-to-end. Обоснование: ранняя интеграция, независимая тестируемость/деплой, минимальный blast radius. Фазы ≠ секции LLD.
R5§5 Phase mapКарта 9 фаз: для каждой — vertical value, depends-on, имя spec-файла. Рекомендуемый порядок 1→2→3→(4,5,6 параллельно)→7→8→9.
R6§6 Global UIДвойной сайдбар (frame C); доменный индекс + дерево статей; мета-секции под статьёй + Edit/Preview-табы (409→reload-баннер); Query и Chat — отдельные экраны (кэш только у Query); full-canvas граф + slide-in drawer; одностраничные настройки + TOC; jobs-drawer (SSE); setup-wizard; UI i18n (ui_language RU/EN независимо).
R7§7 Cross-cuttingСлоение конфига (LLD §10); секреты (argon2, hashed API-keys, Fernet для LLM-ключа/Langfuse); API-конвенции (LLD §8: сессии в Redis/Bearer, RBAC, CSRF, RFC 9457, cursor-пагинация, SSE); harness safety (LLD §4: allowlist, write-scope, schema-validated, data-not-instructions, лимиты, audit_log); политика embedding dim (managed ALTER+reindex); тестирование (LLD §11); observability (счётчики в точке сборки, экспортёр в фазе 9).
R8§8 DeferredBacklog/out-of-v1 (LLD §13): rate limiting, idempotency, soft-delete, reranking, resumable ingest, graph pagination perf, scheduled jobs; per-domain ACL, config hot-reload, webhooks, quality eval. Ни одна фаза их не реализует — перечислены, чтобы оставить дешёвые seams.
R9§9 How to consumeКаждая spec-фаза самодостаточна и готова к writing-plans: Phase 1 spec → writing-plans → implement → verify acceptance → next phase. Спеки ссылаются на LLD за DDL/контрактами, не дублируют их.
