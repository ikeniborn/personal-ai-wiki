1. Резюме спецификации

Цель фазы: docker compose up поднимает весь стек; админ проходит first-run мастер, логинится, создаёт домен, загружает markdown-источник, вручную пишет статью и видит её отрендеренной. LLM нигде не используется.

2. Схемы решения и зависимостей

2.1 Схема решения (компоненты и потоки)

C4-вид: deploy-стек (traefik/api/worker/postgres/redis/init) ⊃ модули пакета paw. Стрелки потоков описаны под диаграммой.

Потоки: браузер → traefik → api (web/UI + /api/v1); api → services → db/repos → postgres; api → storage(PostgresStorage) → postgres (blob bytea / Large Object); api → redis (сессии, CSRF); init → postgres (миграции, до старта api/worker); worker ↔ redis (arq очередь, heartbeat).
