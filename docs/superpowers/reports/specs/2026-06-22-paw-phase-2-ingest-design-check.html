1. Резюме спецификации

1.1 Требования (по секциям)

СекцияТребование
In scopeProviders — ChatProvider/EmbeddingProvider/VisionProvider Protocols; OpenAICompatProvider; structured() с repair-loop; JSON-mode fallback при отсутствии tool-calling; connection/модели из app_settings.
Harness — loop.py (chat→tool_calls→results до final/max_steps), tools.py (read+write), ops/{ingest,init}, versioned prompts/.
Guards — tool-allowlist на op, write-scope target.domain_id == ctx.domain_id, schema-validation до записи, sources/tool-results «data, not instructions», лимиты max_steps/max_tool_calls/max_writes/token-budget/timeout/loop-detection, каждый tool-call → audit_log.
upsert_article идемпотентен по slug (merge); report_issue collect-only, не используется до Phase 6.
Loaders — md/txt passthrough, pdf pymupdf, docx mammoth, html trafilatura → markdownify. (epub/url/image-OCR → Phase 9.)
Ingest pipeline A topic extraction (windowed map-reduce, merge/dedup) → B drafting (create|merge, headings до ##) → C deterministic write → D links → E chunking + embedding.
Write C — articles + article_revisions origin=ai, entities, article_entities, citations.
Links D — typed link_suggestions + co-occurrence по общим article_entities ≥ hub_threshold → related.
Chunking — summary chunk (ord=0, копия в articles.summary); split по ##; semantic split (sentence-embedding breakpoint, bounded target_size); overlap 1–2 sentences; heading_path; chunk_entities.
Vector — vector/embed.py: batch embed → chunks(kind, ord, heading_path, text, tsv, embedding, embedding_version).
Graph writes — graph/repo.py: entity/link upserts (traversal → Phase 3).
Jobs / worker — ingest_domain task; lifecycle queued→running→terminal; progress через Redis pub/sub job:{id} + replay из jobs.log; cancel_requested; heartbeat + reconciler; job-lock per domain; model-lock lock:model:{name}; arq queues (LLM vs light), retries/backoff, poison→dead-letter.
DB additions — entities, article_entities, links, citations, chunks (+HNSW vector_cosine_ops, GIN на tsv, индекс на embedding_version), chunk_entities, jobs; managed embedding-dim migration (vector(dim) + HNSW); смена dim = ALTER + rebuild + reindex (reindex job → Phase 6).
APIPOST /domains/{id}/ingest → job_id; POST /domains/{id}/init (sync: план → topics → per-topic jobs); POST /domains/{id}/sources расширен на pdf/docx/html; GET /jobs/{id} · /events (SSE) · POST /jobs/{id}/cancel.
Web UIDomain Ingest action → job drawer (progress bar + live log SSE + cancel); статья на /articles/{id}; /settings Connection + models + dim (предупреждение ALTER+reindex при смене dim).
ConfigConnection+models+dim в app_settings (key — Fernet); wiki-defaults: языки, chunk-params, hub_threshold, agent-лимиты, link_types, timeouts/retries; per-domain overrides в domains.config.
SecurityPrompt-injection defenses (delimiters, write-scope, max_steps, schema-validation); key расшифровывается только в call-site, не в agent context; per-call audit; upload guard на pdf/docx/html (magic-byte + size); zip/anti-zip-bomb → Phase 9.
Out of scopeRetrieval/query (3) · chat (4) · graph viz (5) · lint/fix/format/reindex (6) · query-cache (7) · MCP (8) · epub/url/image-OCR, bulk zip, observability (9); search_wiki → Phase 3.

1.2 Success Criteria (verifiable)

#Критерий приёмки
AC-1Wizard сохраняет connection+models+dim; managed migration создаёт vector(dim)+HNSW; api/worker подхватывают зашифрованный key (не логируется).
AC-2Upload фикстуры pdf/docx/html/md → ingest → ровно одна статья: article_revisions origin=ai, ≥1 entity, ≥1 citation, chunks вкл. ord=0 summary, embeddings; articles.summary заполнен.
AC-3Co-occurrence linker создаёт related выше порога; LLM link_suggestions — typed links только внутри домена (cross-domain отклонён).
AC-4Прогресс job стримится по SSE и реплеится из jobs.log на reconnect; cancel mid-run — без partial-статьи (cleanup).
AC-5Domain job-lock блокирует второй пишущий job на домен; model-lock сериализует one-model LLM-вызовы.
AC-6Structured-output repair восстанавливается из одного malformed-ответа (stub-LLM); модель без tool-calling — JSON-mode fallback.
AC-7Empty/garbage source → job падает чисто с ошибкой в jobs.error, без статьи.
