Проверка спецификации — Phase 4: Chat + history

paw (Personal AI Wiki) · 2026-06-22-paw-phase-4-chat-design.md · прогон 2026-06-23

Цель спецификации: многоходовой диалог с доменом с сохранением истории; каждый ход переиспользует retrieval-путь Phase 3 и возвращает ответы с цитатами. Сессии перечисляются, титулуются, удаляются и чистятся по per-user retention.

1. Резюме спецификации

Требования (по секциям)

IDСекцияТребование
R1In scope / Chat op harness/ops/chat.py + chat prompt → ChatTurn{answer_md, refs[]}; read-only tools (allowlist как у query); контекст треда = последние history_depth ходов; retrieval = Phase 3 (hybrid + BFS + assembly).
R2In scope / DB chat_sessions (user_id, domain_id, title, last_active_at, индекс (user_id, last_active_at DESC)); chat_messages (session_id, role, content, meta jsonb = refs/citations/model/prompt_version/tokens, индекс (session_id, created_at)).
R3In scope / Services services/chat.py — создать сессию; добавить user/assistant сообщения; auto-title из первого хода; обновлять last_active_at.
R4In scope / API POST /chat (+ SSE стрим ответа); GET /chat/sessions (cursor, свои треды по last_active_at); GET /chat/{session} (сообщения); DELETE /chat/{session}.
R5In scope / Web UI Отдельный экран-мессенджер Chat (frame C); вторичный сайдбар = история сессий (свои, по last_active_at, удаляемые) + поток ходов + ввод; ответ ассистента стримится через SSE; source chips на ход.
R6In scope / Retention users.chat_prefs overrides (retention.max_sessions, retention.max_age_days, history_depth; null → global default); jobs/tasks.py:gc_housekeeping — chat retention (прунинг сессий сверх N / старше N дней, per user); в v1 запуск админом.
R7Security Own-thread isolation (только свои сессии list/read/delete, enforced в deps); контент чата недоверенный в промптах (delimiters); нет write-tools в чате.

Success Criteria (Acceptance, verifiable)

IDКритерий
AC1Многоходовой обмен держит контекст до history_depth; ход N+1 ссылается на ход N.
AC2Первый ход авто-генерирует title; last_active_at обновляется на каждом ходу.
AC3GET /chat/sessions возвращает только сессии вызывающего, newest-active first, по курсору; чужой session id → 404/403 на read/delete.
AC4Ходы ассистента стримятся через SSE и несут refs в meta.
AC5gc_housekeeping чистит сессии сверх max_sessions и старше max_age_days; дефолты при null chat_prefs.
