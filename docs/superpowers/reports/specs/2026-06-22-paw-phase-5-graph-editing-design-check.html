1. Резюме спецификации

Спека описывает Phase 5 — Graph + editing: навигацию по wiki через структуру ссылок. Полноэкранный граф показывает подграф вокруг любой статьи; страница статьи раскрывает реальные backlinks, related-ссылки, цитаты и ревизии; вторичный сайдбар превращается в дерево parent/child. Решение строится на данных link/entity/citation, которые производит Phase 2. Зависит от Phase 2. Изменений схемы БД нет.

Требования (по секциям)

СекцияТребование
In scope · Graph read (LLD §6/§9)graph/repo.py — subgraph-запрос вокруг root, ограниченная глубина, фильтр по типу ссылки; GET /graph?domain=&root=&depth= возвращает nodes + типизированные edges.
In scope · Graph UI (frame 🕸)Полноэкранный Cytoscape (vendored) + тонкие верхние контролы (выбор root, слайдер глубины, фильтр типов related/parent/child/references/depends_on); клик по узлу → slide-in drawer (summary + ссылки + open).
In scope · Link-aware article page (LLD §9)Заполнить метаданные-секции (плейсхолдеры Phase 1): Citations/Sources (citations→sources), Backlinks (links по dst_article_id), Related/parent/child (links по типу), Revisions (article_revisions + rollback); [[refs]] рендерятся как внутренние wiki-ссылки.
In scope · Secondary sidebar tree (LLD §9)Заменить плоский список статей Phase 1 деревом parent/child из links (тип parent/child) с фильтр-боксом.
In scope · EditingПереиспользовать Edit/Preview вкладки Phase 1 + optimistic-lock 409 + rollback; правки, меняющие links/citations, отражаются в метаданных и графе.
Out of scopeLint/fix, меняющие граф (Phase 6); query-cache stale-on-edit (Phase 7); graph pagination perf (backlog); новых таблиц нет.
Data model touchedЧитает links (+ индекс links(dst_article_id)), citations, entities, article_revisions из Phase 2. Без изменений схемы.
Config (LLD §10)bfs_depth/дефолтная глубина графа, allowlist типов ссылок (display + filter), per-domain overrides.
SecurityRead-эндпоинты ограничены доменным доступом вызывающего; рендер статьи + summary санитизируются (nh3); rollback пишет новую ревизию (origin сохранён/аудит), не деструктивная перезапись.

Success Criteria (Acceptance criteria, verifiable)

#Критерий приёмки
AC#1GET /graph возвращает subgraph вокруг root в пределах глубины; edges несут тип; фильтр типов убирает отфильтрованные edges.
AC#2Graph UI рендерит subgraph; слайдер глубины перезапрашивает; клик по узлу открывает drawer с summary + ссылки; «open» ведёт к статье.
AC#3Страница статьи показывает реальные backlinks (обратная сторона исходящей ссылки), related/parent/child ссылки, цитаты со ссылками на их sources, и список ревизий.
AC#4Вторичный сайдбар показывает дерево parent/child; фильтр сужает его.
AC#5Rollback восстанавливает прежнюю ревизию как новую ревизию; optimistic-lock 409 сохраняется.
